# ADR-001: v0.devによるUIプロトタイピング採用

## 背景

- jam-matchのMVP UIを効率的に構築する必要があった
- shadcn/ui + Tailwind CSSベースのコンポーネントが必要
- デザイナー不在の個人開発であり、デザインツールを使ったワークフローは非効率

## 決定内容

- v0.dev（Vercel公式AIツール）でUIプロトタイプを生成し、プロジェクトに取り込む方式を採用
- v0 CLIは利用せず、コードコピー方式で運用（`docs/design/v0_pasted.md`経由）
- 生成されたコードはそのまま使わず、プロジェクト構成に合わせて調整（レイアウト重複除外、API互換性対応等）
- プロンプトは`docs/design/v0-prompts.md`で管理し再現可能にする

## 代替案

| 案 | 却下理由 |
|---|---------|
| Figmaでデザイン → 手動コーディング | 工数が大きい。デザイナー不在では非効率 |
| shadcn/uiのサンプルを手動カスタマイズ | 可能だが、画面全体のプロトタイプを作るには時間がかかる |
| v0 CLI自動連携 | `components.json`の`style: "base-nova"`未対応、chat URLからのID解決不可等の問題で断念。将来改善される可能性あり |

## 結果

- 8画面のUIコンポーネントを1セッションで生成・取り込み完了
- ビルド全通過
- コピペ方式は手動だが、サブエージェント委任で十分効率的
- `ToggleGroup`等の一部APIは手動調整が必要だった（plain buttonで代替）
- 生成コードの品質は高く、微調整のみでプロダクション利用可能
- `globals.css`のテーマ変数は初回取り込み時に統一し、以降は固定する運用とした
